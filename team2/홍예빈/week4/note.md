## 중요! 스프링의 주요 원리 4가지
### 1. IoC(Inversion of Control) : 제어의 역전
    inverse : 정반대로 뒤집힘
    control : 제어
    객체에 대한 제어권을 정반대로 뒤집음
    원래 개발자가 가지고 있던 제어권을 스프링에게 줌
    -> 개발자가 가지고 있는 제어권이 역전 = 개발자의 제어권을 스프링에게 줌
    -> 스프링의 없던 제어권이 역전 = 스프링이 제어권을 가지게 됨
### 2. 컨테이너
    스프링이 객체를 생성해서 쉽게 관리하기 위해 박스를 준비 -> 컨테이너
    스프링 컨테이너 = IoC 컨테이너 = **DI 컨테이너** 
### 3. 스프링 빈
    컨테이너 안에 있는 객체를 지칭
    스프링의 관리를 받는 객체
### 4. DI(Dependency Injection) : 의존성 주입
    dependency : 사용할(하는) 것
    개발자 : 주입당함(받음)
    스프링 : 주입함
    -> 개발자는 객체를 스프링으로부터 주입받음(객체의 제어권이 스프링에게 있기 때문)
    cf. 객체를 사용함 : 객체를 생성해서 메소드/필드를 호출, 접근함
    **"IoC를 구현하기 위해 DI가 필요"**
